HTML over with Turbo

28-12-2020 | ruby rails turbo

Turbo

เป็นอีกเทคนิค และเครื่องมือหนึ่งที่ถูกพัฒนาออกมาเพื่อเพิ่มความเร็วให้กับเว็บของเราในขณะที่ submit form โดยแบ่งความซับซ้อนในหน้าเว็บออกเป็นคอมโพเนนต์เช่นเดียวกับการเขียนแบบ SPA และนอกจากนี้ยังสามารถสตรีม partial page ผ่าน Websocket โดยที่เราไม่จำเป็นต้องเขียน JavaScript เลย

สามารถเข้าไปศึกษาวิธีการใช้ Turbo ได้จากที่นี่ หรือจะดูจากวิดีโอด้านล่างก็ได้นะครับ

สำหรับความคิดเห็นส่วนตัวนั้นต้องบอกเลยว่าชอบเทคนิคอันนี้มาก เนื่องจากโดยส่วนตัวไม่ชอบการเขียน JavaScript และเลือกที่จะใช้ HTML มากกว่า JSON Object

ในบทความนี้จะขอเล่าผ่านตัวอย่างแอพ Todo ที่สร้างขึ้นมา ซึ่งมีโมเดล Todo ในการเก็บงานสิ่งที่ทำ (task:string) และ flag ว่าทำเสร็จแล้วหรือไม่ (done:boolean) โดยตัวแอพจะมี form สำหรับให้เราบันทึก task ที่จะทำ จากนั้น task ก็จะถูกสร้างเก็บไว้ในฐานข้อมูล และแสดงผลอยู่บนบัญชีงานที่อยู่ด้านล่าง โดยแต่ละงาน เราสามารถที่จะกด Done เพื่อเสร็จสิ้นงาน และกด Delete เพื่อลบออกจากฐานข้อมูลได้ สำหรับ Turbo จะทำให้แอพของเรามีการ sync หน้าเว็บในกรณีที่เราเปิดหลายแท็บ หรือวินโดวส์ผ่าน Websocket

app/views/todos/index.html.erb

<p id="notice"><%= notice %></p>

<h1>Todos</h1>

<%= turbo_stream_from :todos %>

<%= turbo_frame_tag :new_todo do %>
  <%= render 'form', todo: @todo %>
<% end %>

<%= turbo_frame_tag :todos do %>
  <%= render @todos %>
<% end %>

app/models/todo.rb

class Todo < ApplicationRecord
  after_create_commit -> { broadcast_append_to :todos, partial: 'todos/todo', locals: { todo: self } }
  after_update_commit -> { broadcast_action_to :todos, action: "replace", target: "#{self.class.name.downcase}_#{id}", partial: 'todos/todo', locals: { todo: self } }
  after_destroy_commit -> { broadcast_action_to :todos, action: "remove", target: "#{self.class.name.downcase}_#{id}" }
end

โดยหลักๆ ของแอพจะใช้

  • Turbo Frames ในการแยกคอมโพเนนต์ และจัดการแบบอิสระต่อกัน
  • Turbo Streams ในการรับ partial page ผ่าน HTML และอัพเดตไปยังคอมโพแนนต์ที่กำหนดไว้
